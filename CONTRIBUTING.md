## How Can I Contribute?

### Reporting Bugs

If you find a bug, please open an issue in the repository.

Include:

- Clear description of the problem
- Steps to reproduce the issue
- Expected vs actual behavior
- Screenshots or logs (if available)

---

### Suggesting Enhancements

We welcome ideas and improvements!

When submitting a suggestion, please include:

- What feature or improvement you want
- Why it would be useful
- Any examples or references if possible

---

### Pull Requests

We love pull requests. To contribute code:

1. Fork the repository
2. Create a new branch with a meaningful name
3. Make your changes
4. Ensure your code works and does not break existing features
5. Use clear and descriptive commit messages
6. Push your branch to your fork
7. Open a pull request to the `main` branch

Make sure your code follows the project’s style guidelines.

---

## Style Guidelines

To keep the codebase clean and consistent:

- Use proper indentation (spaces preferred)
- Keep code readable and well-structured
- Limit long lines (80–120 characters recommended)
- Use meaningful variable and function names
- Add comments where necessary
- Follow existing project patterns
